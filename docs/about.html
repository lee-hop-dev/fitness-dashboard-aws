Daily Data Flow

06:00 UTC
EventBridge fires scheduled trigger
AWS EventBridge rule invokes the FitnessDashboardCollector Lambda on a fixed cron schedule, bypassing any manual interaction.
06:00 + ~30s
Lambda authenticates with Intervals.icu
Lambda retrieves the API key from Secrets Manager at runtime — never stored in code. Uses Basic Auth to call the Intervals.icu API (the authoritative data source for all training metrics).
06:01
Activities, wellness, and curves written to DynamoDB
Three separate DynamoDB tables updated: Activities (training sessions), Wellness (daily CTL/ATL/TSB/HRV), and Curves (power, pace, and HR curves). On-demand billing — no idle cost.
06:02
Static JSON files written to S3
Processed curve data written as static JSON files to S3 for efficient frontend consumption, reducing DynamoDB read costs at page load.
Browser request
CloudFront serves assets; API Gateway handles data calls
HTML/CSS/JS served from CloudFront CDN (S3 origin). All live data calls from the browser hit API Gateway, which routes to the appropriate DynamoDB query Lambda.

Serverless compute for all backend logic — data collection, OAuth token exchange, and DynamoDB query functions. Zero idle cost; only billed per invocation.

DynamoDB
Activities · Wellness · Curves (on-demand billing)
Three NoSQL tables with on-demand capacity — no provisioned throughput cost. Stores all training activities, daily wellness metrics, and power/pace/HR curves.
☁ CDN / Hosting
CloudFront + S3
E2A1SYDA1ZW3KS · fitness-dashboard-frontend-656370357696
CloudFront CDN distributes the static frontend globally from an S3 origin. All assets cached at edge — fast load times regardless of visitor location.
🔌 API
API Gateway
j2zxz92vd4.execute-api.eu-west-2.amazonaws.com/prod/
REST API exposing six endpoints to the frontend — activities, wellness, athlete profile, power curve, pace curve, and HR curve. CORS configured for CloudFront domain.
🔐 Security
Secrets Manager
fitness-dashboard/intervals-key · strava-credentials
All API credentials (Intervals.icu key, Strava client ID/secret/refresh token) stored in Secrets Manager. Retrieved at Lambda runtime — never in source code or environment variables.
⏰ Scheduling
EventBridge
Daily cron — 06:00 UTC
Managed event bus rule fires the collector Lambda every morning at 06:00 UTC. Fully managed — no servers, no cron jobs, no maintenance overhead.
📊 Monitoring
CloudWatch
7 alarms · Lambda errors · API 5xx · DynamoDB throttles
Seven CloudWatch alarms covering Lambda error rates, throttles, API Gateway 5xx responses, and DynamoDB throttling events. Real-time alerting on infrastructure health.
🏗 Infrastructure as Code
AWS CDK
7 stacks · TypeScript · CloudFormation-backed
All infrastructure defined in code across seven CDK stacks. Repeatable, version-controlled deployments. No manual console clicking — every resource is declarative and auditable.
💰 Cost Control
Budgets + Kill Switch
4-tier alerts · Emergency shutdown Lambda
Four-tier AWS Budget alerts from $2.40 to $10. An emergency kill-switch Lambda can disable EventBridge, scale DynamoDB to zero, and pause CloudFront via a single POST request.

06

UNDER THE HOOD

The visible dashboard is the thin layer on top. What a visitor doesn't see is where the engineering complexity lives — the systems that run silently every day to keep the data fresh, the infrastructure correct, and the costs controlled.

🔄
Automated Daily Sync
Every morning at 06:00 UTC, EventBridge fires the collector Lambda without any human interaction. It fetches up to 400 days of activity data, daily wellness metrics, and multi-type performance curves — all written to DynamoDB and S3 automatically.
EventBridge · Lambda · DynamoDB · S3
🔐
Zero Hardcoded Credentials
No API keys, tokens, or secrets exist anywhere in the codebase or environment variables. Every credential — Intervals.icu API key, Strava client secret, OAuth refresh tokens — is retrieved from Secrets Manager at Lambda runtime and discarded after use.
Secrets Manager · IAM · Least Privilege
📐
Infrastructure as Code — All 7 Stacks
Every AWS resource — tables, Lambdas, API routes, alarms, budgets, even the kill switch — is defined in TypeScript CDK. The entire platform can be torn down and redeployed from source in minutes. No manual console configuration exists.
AWS CDK · CloudFormation · TypeScript
📊
7 Live CloudWatch Alarms
Lambda error rates, Lambda throttles, API Gateway 5xx responses, and DynamoDB read/write throttles are all monitored in real time. Alarms are wired into an operational dashboard — any infrastructure fault surfaces immediately without log-diving.
CloudWatch · Alarms · OpsHub Dashboard
💀
Emergency Kill Switch
A dedicated Lambda accessible via a single POST request can shut down the entire platform: disables the EventBridge schedule, scales DynamoDB to zero, and pauses CloudFront distribution. Designed to prevent runaway costs if something goes wrong.
Lambda · EmergencyShutdownStack · Budgets
🏃
Authoritative Data Discipline
All training metrics — FTP, power curves, running PBs, CTL/ATL/TSB — are fetched directly from Intervals.icu APIs. No metrics are ever calculated from raw activity data in the application layer. This is a hard architectural rule maintained across every phase.
Intervals.icu · API-first · No derived metrics
